# Claude Code ガイド

## セッション開始時にやること
新しいセッションを開始したら、必ず以下を読んでから作業を始めること：
- `~/.claude/projects/[project-path]/memory/MEMORY.md`（インデックス）
- インデックスに記載された関連ファイル
- `.claude/PROJECT_CONTEXT.md`（プロジェクト固有の仕様・パス・制約）

## セッション終了時にやること
「今日はここまで」「ここまでにする」など作業終了を示す発言があったら必ず以下を実行すること：
1. gitコミット用サマリー（1行）とディスクリプション（箇条書き）を出力する
2. セッションで決定した設計・仕様などをメモリファイルに記録・更新する

## 作業ルール
- 作業対象ディレクトリ・禁止ディレクトリは `.claude/PROJECT_CONTEXT.md` を参照すること
- ゲームデザイン仕様はプロジェクトの設計書を正とする
- 1コンポーネント150行超禁止（超える場合は分割案を提示してから実装）
- 既存テストが通らない実装は提出しない

## ゲームディレクター監修ルール（必須）
コードの実装・変更を伴う依頼は、必ず `game-director` エージェントに委譲する。
game-director が調査・承認・実装・最終チェックを完結させる。Claude Code は結果を受け取ってユーザーに報告するだけ。

### 実装前の理解度確認（必須・省略禁止）
game-director に渡す前に、必ず `AskUserQuestion` ツールでユーザーに確認問題を出すこと。
ユーザーが正しく理解していることを確認してから game-director に指示を渡す。

1. 「これから何をするか」をユーザーが正しく理解しているか問う
2. 回答内容を整理・要約して game-director への指示に変換する
3. 理解が不十分な場合は補足説明し、再度確認する

```javascript
Agent({
  subagent_type: "game-director",
  prompt: "【依頼】[ユーザーの依頼と確認済み内容をまとめて記載]"
})
```

**例外**: ドキュメント・メモリファイルの更新のみの場合は省略可

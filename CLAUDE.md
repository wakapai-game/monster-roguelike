# Claude Code ガイド

## セッション開始
- MEMORY.md インデックスに記載された関連ファイルを読む
- `.claude/PROJECT_CONTEXT.md`（仕様・パス・制約）を読む

## セッション終了（「今日はここまで」等）
1. gitコミット用サマリー（1行）＋ディスクリプション（箇条書き）を出力
2. 決定した設計・仕様をメモリファイルに記録・更新
3. `/compact` の実行を促す

## 作業ルール
- ディレクトリ制約は `PROJECT_CONTEXT.md` 参照
- ゲームデザイン仕様は設計書を正とする
- 1コンポーネント150行超禁止（超える場合は分割案を提示してから実装）
- 既存テストが通らない実装は提出しない
- CLAUDE.md は100行以内に維持（超える場合は先に圧縮・整理）
- 実装・仕様変更後は必ず `DESIGN.md` を更新（プロジェクト再起動時のコンテキスト源）

## ゲームディレクター監修（必須）
コード実装・変更は必ず `game-director` に委譲。Claude Code は結果を受け取り報告するのみ。

**委譲前に必須**: `AskUserQuestion` でユーザーの理解を確認する（何をするか → 整理・要約 → 不十分なら補足・再確認）。

```javascript
Agent({
  subagent_type: "game-director",
  prompt: "【依頼】[確認済みの依頼内容]\n【必須】完了後に DESIGN.md を更新すること"
})
```

**例外**: ドキュメント・メモリのみの更新は省略可

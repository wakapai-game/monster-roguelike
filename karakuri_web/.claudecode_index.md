# karakuri_web ファイルインデックス

命名規約: `[CATEGORY]_[Module]_[Action].js` — 詳細は `.claude/NAMING_CONVENTION.md`  
新規ファイル追加時はこのファイルを更新すること（logic/ui/data-engineer のセルフチェック必須項目）

---

## DATA（マスターデータ・グローバル状態）

| ファイル | 内容 |
|---|---|
| `DATA_Game_Master.js` | 全マスターデータ（TECH_PARTS / STAT_PARTS / OPTION_PARTS / KARAKURI_DATA / JUMA_DATA / ENEMY_DATA / アイテム / クエスト） |
| `DATA_App_State.js` | グローバル appState（セーブ状態・パーティ・マップ進行・所持品） |

## LOGIC（純粋ロジック・クラス）

| ファイル | 内容 |
|---|---|
| `LOGIC_Battle_Core.js` | Karakuri / Timeline クラス・BattleEngine ラッパー・findTechPart 等 |
| `LOGIC_Battle_Engines.js` | EngineCase1-4・KarakuriEngine の全実装 |
| `LOGIC_Map_Generator.js` | TutorialMapGenerator / MapGenerator |
| `LOGIC_Save_Persistence.js` | saveGame / loadGame / deleteSave / hasSaveData |
| `LOGIC_Audio_BGM.js` | BGM トラック定義・再生制御・画面遷移連動 |
| `LOGIC_Sprite_Generator.js` | プロシージャルスプライト生成（Canvas procedural pixel art） |

## UI ルート

| ファイル | 内容 |
|---|---|
| `app.js` | エントリポイント — 全画面・ゲームループ統合制御 [entrypoint例外] |
| `UI_Title_Art.js` | タイトル画面 Canvas ロゴ・背景アート |

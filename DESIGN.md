# Monster Rogue - Game Design Document

「スタミナを装甲とし、タイムラインを支配せよ。」

Monster Rogue は、スタミナ管理とタイムライン（行動順）を駆使して最上階を目指す、タクティカル・ローグライクRPGです。

---

## 🔹 ゲームの基本システム

### 1. 攻防フェーズ式ATB（アクティブ・タイム・バトル）
戦闘は独自の**リアクション型バトルシステム**で進行します。
画面中央の「ACTION QUEUE」で敵味方の行動順を確認でき、スピード（SPD）が高いモンスターほど順番が早く回ってきます。

- **攻撃フェーズ (自分のターン)**：
  自身のATBゲージがMAXになった際に発生します。「攻撃・バフ・罠」系の技を使用して敵を攻め立てるか、控えモンスターと交代することができます。（※このフェーズではアイテムは使用できません）
- **防御フェーズ (敵のターン)**：
  敵のATBゲージがMAXになり、こちらに攻撃を仕掛けてくる直前に**ゲームが一時停止**し、防御メニューが開きます。敵の予告技を確認した上で、「防御系の技」を使う、「アイテム」で回復する、控えモンスターを身代わりに出す、あるいは**「身を守る（ダメージ・ST消費を半減）」**といったリアクションを選択して生き残りを図ります。

## 🔹 ステータスシステム

### 固有ステータス (Base Stats)
すべてのモンスターが持つ基本ステータスです。えさによって直接加算されます。

- **HP**: 体力。0になるとそのモンスターは戦闘不能になります。
- **ST (Stamina)**: スタミナ兼装甲。攻撃を受けると減少し、0になるとブレイク状態になります。
- **ATK**: 物理攻撃力（物理系スキルのダメージ倍率）
- **DEF**: 物理防御力（物理ダメージ軽減）
- **MAG**: 魔法攻撃力（魔法系スキルのダメージ倍率）
- **SPD**: 素早さ。ATBゲージの蓄積速度に直結します。

### 可変パラメータ (Variable Params)
えさを与えることで上昇し、最終ステータスに影響するパラメータです。

- **大きさ (Size)**: +5 HP, +2 ST, -0.2 SPD / pt。表示ラベル: SS(-10) / S(-5) / M(0) / L(5) / LL(10)
- **賢さ (Intelligence)**: +1 MAG, +0.5 SPD, -2 HP / pt。Lv.1〜5（4ポイントごとにランクアップ）

## 🔹 バトルの戦略とメカニクス

### 1. BREAK（ブレイク）状態

ST が `0` になると「ブレイク状態」に陥ります。

- **受けるダメージが直接 HP に入ります（通常は ST が盾になります）**
- **ダメージ量が 2.0 倍になります**（属性有利との組み合わせで最大 4.0 倍）
- **技を使うとき、ST の代わりに直接 HP を消費します**（ボタンに「★HP自傷」と表示）

敵がブレイク状態になると、敵のカードが**赤く点滅**します。プレイヤーもブレイクすると自分のカードが赤く点滅します。

> **敵の ST 残量は画面に表示されません。** 攻撃を続けて赤い点滅を確認することがブレイク検知の唯一の手段です。

### 2. BAP（ベースライン攻撃ペナルティ）
すべての攻撃系スキルは命中時、ダメージに加えて**追加の ST ダメージ（10〜30）**を与えます。これにより、防御力が高い相手でも ST を削れます。

### 3. ダメージ計算式
```
物理ダメージ = base_power × (ATK ÷ DEF) × affinity_mult × break_mult
魔法ダメージ = base_power × (MAG ÷ DEF) × affinity_mult × break_mult
```
- `affinity_mult`: 属性有利 2.0倍、不利 0.5倍、通常 1.0倍（主・副属性を両方突くと最大 4.0倍）
- `break_mult`: ブレイク中 2.0倍

### 4. モンスターの属性と相性
火・水・氷・雷・土・風・光・闇の **8属性** が存在します。
スキルボタンに「バツグン」バッジが表示されると効果 2.0倍、「いまいち」で 0.5倍になります。

### 5. 控えとの交代 (Swap)
バトル中、控え（Reserve）にいるモンスターは**ターンが経過するごとに少しずつ ST が自動回復します。**
攻撃フェーズ・防御フェーズのどちらでも即座に控えと交代可能です。

### 6. 身を守る (Guard)
防御フェーズで選択可能。受けるダメージと ST 消費を **半減** します。

## 🔹 スキルシステム

スキルは **category** によって使用できるフェーズが異なります。

| カテゴリ | 使用可能フェーズ | 説明 |
|---------|--------------|------|
| attack | 攻撃フェーズ | 敵に ST/HP ダメージを与える |
| magic | 攻撃フェーズ | 魔法系攻撃（MAG 基準） |
| buff | 攻撃フェーズ | 自身の強化 |
| trap | 攻撃フェーズ | 相手の行動に反応する罠 |
| defense | 防御フェーズ | ダメージ軽減・無効化 |
| heal | 防御フェーズ | 回復系 |

モンスター1体につき技スロットは最大 **4つ** です。インベントリで習得した技を自由にセットできます。

---

## 🔹 バトルアイテムシステム

防御フェーズ中に「アイテム」タブから使用できます。

| アイテム | 効果 |
|---------|------|
| キズぐすり | HP を 50 回復 |
| スタミナドリンク | ST を 50 回復（ブレイク解除にも！） |

アイテムはバトル外（インベントリ画面）でも確認できます。報酬・チュートリアル開始時に入手できます。

---

## 🔹 えさシステム

えさはインベントリ画面でモンスターに与えられます。モンスター **1体につき最大 10回** まで与えられます（`feed_count` で管理）。

### えさの種類（9種）

| えさ名 | 効果（base_stats加算 or params加算） |
|-------|-----------------------------------|
| にく | ATK +5 |
| やさい | DEF +5 |
| さかな | MAG +5 |
| くだもの | SPD +3 |
| おにく | HP +30 |
| まほうくさ | MAG +8, HP -10 |
| おおきなにく | size +3 (HP+15, ST+6, SPD-0.6) |
| かしこいえさ | intelligence +4 (MAG+4, SPD+2, HP-8) → 賢さランクアップ |
| スタミナえさ | max_st +20 |

えさによる変化は**永続的**です。戦闘後も引き継がれます。

## 🔹 ゲームサイクルと進行

1. **スタート（プロローグ）**
   - プレイヤーは名前を登録し、王からスターターモンスター **3体** の卵を受け取ります。
   - 卵から孵ったモンスターは初期パーティに加わります。

2. **チュートリアル選択**
   - フル版・シンプル版・スキップの3択から選択できます。
   - フル版またはシンプル版を選ぶと、チュートリアルバトルが開始されます。

3. **拠点（Stage Hub）**
   - PARTY 画面でパーティ編成・ステータス確認
   - INVENTORY 画面でえさ適用・技の確認
   - ステージ選択（解放済みステージから1つ選ぶ）

4. **マップ探索 (Node Map)**
   - 3つのステージ（始まりの草原・迷いの森・試練の塔）から選択
   - ランダム生成されたノードを進みます（通常・エリート・ボス・休憩ノード）

5. **バトルと報酬**
   - 通常バトルに勝利すると報酬 2つ を獲得（スキル・バトルアイテム・えさからランダム）
   - **ステージクリア後（ボス撃破後）に卵選択画面が出現** → 新モンスターを入手

6. **全滅とローグライク**
   - 全滅するとプログレスがリセットされ GAME OVER となります。

## 🔹 チュートリアルシステム

### モード選択

| モード | 説明 |
|------|------|
| フル版 | ATBループを一時停止し、オーバーレイで詳細解説。OK ボタンで再開 |
| シンプル版 | ヒントバー（#action-menu 上部）に簡潔なテキストを 6 秒表示。ゲームは止まらない |
| スキップ | チュートリアルなし。すぐに拠点へ |

### チュートリアルバトル

専用の敵「修行用ダミー」（`TUTORIAL_ENEMY`）と戦います。
- **属性**: 氷（ice）→ フレイムパピーの fireball が「バツグン」になり属性相性を体験できる
- **ST**: 200（数回の攻撃が必要、ST削り工程を体験）
- **DEF/MAG**: 高め（単純攻撃では倒しにくく、ブレイクの重要性を学べる）

### チュートリアルステップ（9ステップ）

| ステップID | タイミング | 内容 |
|-----------|----------|------|
| `action-queue` | バトル開始時 | ACTION QUEUE の読み方と SPD の説明 |
| `attack-phase` | 攻撃フェーズ初回 | スキル選択・STを削る基本攻撃の説明（敵ST非表示・赤点滅でブレイク確認） |
| `st-chip` | 攻撃してもブレイクしない場合 | ST削り継続の促し |
| `affinity` | 攻撃フェーズ（バツグンスキルあり） | 属性バッジ「バツグン」の説明 |
| `defense-phase` | 防御フェーズ初回 | 防御オプション（身を守る・スキル・アイテム・交代）の説明 |
| `item-use` | アイテムタブ開時 | キズぐすり・スタミナドリンクの使い方 |
| `player-break` | 自分がブレイクしたとき | HP自傷ペナルティと交代・アイテムによる対処法 |
| `swap` | 交代可能なフェーズ | 控えとの交代タイミングと効果 |
| `break` | 敵がブレイクしたとき | ブレイク中のダメージボーナスと追撃の重要性 |
| `reward` | バトル終了後・報酬画面 | 報酬種別（技・アイテム・えさ）の説明。モード問わず常にオーバーレイ表示 |

## 🔹 HELP システム

どの画面からでも右下の **「?」ボタン**（固定ボタン）でいつでも開けるヘルプオーバーレイです。

### 6タブ構成

| タブ | 内容 |
|-----|------|
| バトルの流れ | ATB・攻撃/防御フェーズの基本フロー |
| ST・ブレイク | ST システム、ブレイク倍率、HP自傷ルール |
| 属性相性 | 8属性の有利/不利マトリクス表 |
| 技・アイテム | スキルカテゴリ、バトルアイテム一覧 |
| 育成・強化 | えさシステム、size/intelligence パラメータ |
| マップ・用語集 | ノードタイプ、用語集 |

HELP を閉じると元の画面に戻ります（オーバーレイなので画面状態はそのまま）。

---

## 🔹 セーブ / ロード

`localStorage` を使用してゲームデータを自動保存します。

- `saveGame()`: 現在の `appState` をシリアライズして保存
- `loadGame()`: 保存データを読み込み `appState` に復元
- `deleteSave()`: セーブデータを削除（GAME OVER 時等）

スタート画面に「続きから」ボタンが表示され、セーブデータが存在する場合はそこから再開できます。

スタート画面からアクセスできる情報オーバーレイです。

| タブ | 内容 |
|-----|------|

## 🔹 マップシステム

`MapGenerator` クラスが各ステージのノードツリーをランダム生成します。

### ノードタイプ

| タイプ | 内容 |
|------|------|
| 通常 (normal) | 標準的な敵とのバトル |
| エリート (elite) | 強敵バトル。より良い報酬 |
| ボス (boss) | ステージボス。撃破でステージクリア・次ステージ解放 |
| 休憩 (rest) | 回復・アイテム入手イベント（予定） |

### ステージ構成

| ステージ | 名称 | 解放条件 |
|---------|------|---------|
| 1 | 始まりの草原 | 初期解放 |
| 2 | 迷いの森 | Stage 1 ボス撃破 |
| 3 | 試練の塔 | Stage 2 ボス撃破 |
